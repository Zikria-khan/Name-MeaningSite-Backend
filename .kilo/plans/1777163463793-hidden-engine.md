# Plan: Modify Category and Gender Filters

## Goal
Modify the name filtering system to support category queries with 2-3 words (no duplicates) and ensure gender filters are limited to single words.

## Current State Analysis
- Category filter currently accepts a single string and uses regex matching against category arrays
- Gender filter uses word boundary regex matching
- No input validation for multiple words or duplicates in category
- No restriction on gender being single word

## Proposed Changes

### 1. Category Filter Enhancement
- **Input Parsing**: Allow category parameter to contain 2-3 words separated by commas or spaces
- **Validation**: 
  - Split by comma/space
  - Remove duplicates (case-insensitive)
  - Limit to 2-3 unique words
  - Each word must be 2-20 characters, alphanumeric/hyphen/apostrophe only
- **Filtering Logic**: Match names where category array contains ALL specified words (AND logic) instead of simple regex

### 2. Gender Filter Validation  
- **Input Validation**: Ensure gender parameter contains exactly one word
- **Word Requirements**: 2-20 characters, letters/hyphen/apostrophe only
- **Matching**: Keep current word-boundary regex but add input validation

### 3. API Documentation Update
- Update route documentation to reflect new category format (comma/space separated, 2-3 words)
- Clarify gender must be single word

### 4. Error Handling
- Return 400 Bad Request for invalid category (wrong count, duplicates, invalid characters)
- Return 400 Bad Request for invalid gender (multiple words, invalid characters)

### 5. Backward Compatibility
- Single word categories should still work
- Existing single word genders should continue working

## Implementation Steps

1. **Input Parsing Functions**
   - Create `parseCategoryFilter()` function in namesController
   - Create `validateGenderFilter()` function

2. **Controller Updates**
   - Modify `getNamesByReligion()` to use new parsing/validation
   - Update category filtering logic to use AND matching for multiple words

3. **Route Documentation**
   - Update JSDoc comments in names.js routes

4. **Testing**
   - Add tests for new validation logic
   - Test edge cases (duplicates, invalid characters, wrong word counts)

## Files to Modify
- `src/controllers/namesController.js` - Add parsing/validation, update filtering
- `src/routes/api/v1/names.js` - Update documentation
- `test-filters.js` - Add validation tests
